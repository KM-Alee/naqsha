## 1. Project identity and vocabulary

**NAQSHA** (never "NAQSH") is a headless Python agent runtime. It is not a chat-API wrapper; it
is a production-shaped execution engine with enforced contracts, append-only traces, and
structured safety gates.

Use this vocabulary at all times — the codebase, tests, and docs all use these exact terms:

| Correct term                    | Do NOT say                            |
|---------------------------------|---------------------------------------|
| NAQSHA                          | NAQSH                                 |
| Core Runtime                    | platform, app, engine                 |
| Team Workspace                  | agent folder, agent workspace         |
| Hierarchical QAOA Trace         | flat trace, provider transcript       |
| NAP Action (V2)                 | free-form model instruction           |
| Dynamic Memory Engine           | chat history, SimpleMem               |
| Typed Event Bus                 | print statements, logging             |
| Decorator-Driven API            | class-based tools                     |
| Tool-Based Delegation Model     | graph routing, state machine          |
| Automated Rollback Manager      | undo, git revert                      |
| Workbench TUI                   | dashboard, web UI                     |
| Role-Based Tool Policy          | per-agent permissions                 |
| Circuit Breaker                 | retry limit, error handler            |
| Tool Policy / Approval Gate     | prompt-only safety                    |

Docs: https://km-alee.github.io/naqsha/concepts/

## 2. Architecture overview

```
Public API
  naqsha CLI  ──►  AgentWorkbench  ──►  CoreRuntime
  Workbench TUI  ──────────────────────────► RuntimeEventBus (subscribe only)

Core Runtime (headless — never imports tui/)
  CoreRuntime (run loop)
    ├── Typed Event Bus (RuntimeEventBus)
    ├── Tool Policy + Approval Gate
    ├── Budget Meter (steps / tokens / tool-calls / wall-time)
    ├── Circuit Breaker (per-tool consecutive failure tracking)
    └── Tool Scheduler (serial + parallel)

Adapters
  Model Adapters  (OpenAI-compat / Anthropic / Gemini / Ollama / Fake)
  Tool Registry + Decorator-Driven API  (@agent.tool)
  Dynamic Memory Engine  (SQLite WAL, MemoryScope, DDL safelist)
  Hierarchical QAOA Trace Store  (append-only JSONL)

Orchestration
  TeamTopology  (naqsha.toml)
  Tool-Based Delegation  (delegate_to_<worker> tools)
  team_runtime builders

Safety
  Observation Sanitizer  (runs before trace, memory, prompt injection)
  Reflection Loop + Automated Rollback Manager
```

Key constraint: `core/` MUST NEVER import from `tui/`. The core is headless and must be
importable without `textual` installed.

## 3. Module ownership map

Every module has a single owner. Do not reach across these boundaries.

### `src/naqsha/core/`
Headless execution engine.
- `runtime.py`        — CoreRuntime: orchestrates the agent run loop
- `events.py`         — All Pydantic event models for the Typed Event Bus
- `event_bus.py`      — RuntimeEventBus: subscribe, emit, async generator
- `policy.py`         — Tool Policy: allow, deny, approval requirements
- `scheduler.py`      — Serial vs. parallel execution, per-tool timeouts
- `circuit_breaker.py`— Circuit Breaker: consecutive failure tracking
- `budgets.py`        — BudgetMeter: hard caps for steps, tokens, tool calls, wall-clock
- `approvals.py`      — StaticApprovalGate, InteractiveApprovalGate

### `src/naqsha/models/`
Provider translation. Core Runtime only speaks NAP V2.
- `nap.py`            — NAP V2 protocol: NAPAction, NAPAnswer, SpanContext
- `factory.py`        — model_client_from_profile
- `trace_turns.py`    — Hierarchical QAOA Trace → neutral provider transcript
- `openai_compat.py`, `anthropic.py`, `gemini.py`, `ollama.py`  — Thin Adapters
- `trace_replay.py`   — TraceReplayModelClient for deterministic replay
- `errors.py`         — ModelInvocationError
- `http_json.py`      — JSON POST, HTTP error handling, header redaction

Decorator-Driven API, schema generation, execution.

- `context.py`        — AgentContext: stable public API for tool authors
- `starter.py`        — V2 Starter Tool Set (all defined with @agent.tool)
- `decorated_adapter.py` — decorated_to_function_tool: bridges @agent.tool to FunctionTool

### `src/naqsha/memory/`
Dynamic Memory Engine.

## 4. Safety invariants (non-negotiable)

These must hold at every commit. A change that violates these is not done, even if tests pass.

1. All tool output is an **Untrusted Observation**. It informs the model but cannot instruct the
   runtime.
2. The **Observation Sanitizer** runs before every trace write, memory write, and model context
   injection. No raw tool output ever bypasses it.
3. **Budget Limits fail closed.** Exhausted budgets produce structured RunFailed events, not
   warnings or soft stops.
4. **`auto_merge = false` is the default everywhere.** Opt-in only.
5. The **Reliability Gate** is mandatory before any Reflection Patch merge, whether auto or human.
6. **Worker isolation is absolute.** No AgentContext leaks from Orchestrator to Worker.
7. **`core/` never imports from `tui/`.**
8. The **DDL safelist is enforced.** DROP TABLE, DROP INDEX, destructive DDL via Memory Schema
   Tool are always rejected with ForbiddenDDLError.
9. **Credentials are environment variable names in config files, never secret values.**
10. **Private memory namespaces are agent-scoped and inaccessible to other agents at the SQL
    level** — not just application logic.

```bash
# Core runtime only (no optional deps)
pip install naqsha

# With Workbench TUI (Textual + Rich)
pip install "naqsha[tui]"

# With memory embeddings (sqlite-vec)
pip install "naqsha[memory]"

# Everything
pip install "naqsha[tui,memory]"

# Developer setup (from a clone)
git clone https://github.com/KM-Alee/naqsha.git
cd naqsha
uv sync --extra dev

# Run tests (no API keys needed — fake model client)
uv run --extra dev pytest

# Lint (must be zero errors)
uv run --extra dev ruff check .

# Build docs
uv run mkdocs build --strict
```

Required Python: >=3.11
Package manager: uv (preferred) or pip
Test runner: pytest with fake model clients — no API keys ever required for the test suite.

## 7. Core Runtime

### Minimal library usage

```python
from naqsha import build_runtime, load_run_profile

runtime = build_runtime(load_run_profile("local-fake"))
result = runtime.run("What is 2 + 2?")
print(result.answer)   # "4"
print(result.failed)   # False
```

### With the Typed Event Bus

```python
from naqsha import build_runtime, load_run_profile, RuntimeEventBus
from naqsha.core.events import ToolInvoked, RunCompleted

bus = RuntimeEventBus()

@bus.subscribe
def on_tool(event: ToolInvoked):
    print(f"tool called: {event.tool_name}")

@bus.subscribe
def on_done(event: RunCompleted):
    print(f"answer: {event.answer}")

runtime = build_runtime(load_run_profile("local-fake"), event_bus=bus)
runtime.run("ping")
```

## 8. Decorator-Driven API (@agent.tool)

### Minimal tool

```python
from naqsha.tools import agent, AgentContext

@agent.tool(risk_tier="read", description="Return current UTC time.")
def clock(ctx: AgentContext) -> str:
    from datetime import datetime, UTC
    return datetime.now(UTC).isoformat()
```

### Risk tiers

| Tier          | Meaning                            | Default gate                        |
|---------------|------------------------------------|-------------------------------------|
| `read`        | Read-only; no side effects         | No approval required                |
| `write`       | Writes data or state               | InteractiveApprovalGate in TTY mode |
| `side-effect` | External side effects              | Requires explicit approval          |

### Supported type hints → JSON Schema

| Python type              | JSON Schema                             |
|--------------------------|-----------------------------------------|
| `str`                    | `string`                                |
| `int`                    | `integer`                               |
| `float`                  | `number`                                |
| `bool`                   | `boolean`                               |
| `Optional[T]`            | nullable T                              |
| `list[T]`                | array with item schema                  |
| `dict[str, T]`           | object with additionalProperties        |
| Pydantic `BaseModel`     | object with full Pydantic schema        |
| `AgentContext`           | injected at runtime; omitted from schema|

`AgentContext` parameters are auto-injected and never exposed to models.
Malformed signatures raise `ToolDefinitionError` at decoration time — not at runtime.

### Complex params with Pydantic

```python
from naqsha.tools import agent, AgentContext
from pydantic import BaseModel, Field

class SearchParams(BaseModel):
    query: str = Field(description="Search query.")
    limit: int = Field(default=10, ge=1, le=100)

@agent.tool(risk_tier="read", description="Search knowledge base.")
async def search_kb(params: SearchParams, ctx: AgentContext) -> list[dict]:
    scope = ctx.shared_memory
    if scope is None:
        return []
    rows = scope.query(
        "SELECT title, snippet FROM shared_articles WHERE snippet LIKE ? LIMIT ?",
        (f"%{params.query}%", params.limit),
    )
    return [{"title": r[0], "snippet": r[1]} for r in rows]
```

### AgentContext fields

| Field             | Type                  | Description                                     |
|-------------------|-----------------------|-------------------------------------------------|
| `agent_id`        | `str`                 | Current agent identifier                        |
| `run_id`          | `str`                 | Unique run identifier                           |
| `workspace_path`  | `Path \| None`        | Workspace root; None for in-memory runs         |
| `shared_memory`   | `MemoryScope \| None` | Team-wide memory (shared_* tables)              |
| `private_memory`  | `MemoryScope \| None` | Agent-private memory (private_<agent_id>_*)     |
| `span`            | `Span \| None`        | Active trace span for custom metric recording   |

### Starter Tool Set (naqsha.tools.starter)

| Tool                  | Risk tier   | Description                                      |
|-----------------------|-------------|--------------------------------------------------|
| `clock`               | `read`      | Return current UTC time                          |
| `list_files`          | `read`      | List files in a directory                        |
| `read_file`           | `read`      | Read a file's contents                           |
| `write_file`          | `write`     | Write content to a file                          |
| `memory_schema`       | `write`     | Execute DDL schema changes (safelist enforced)   |
| `list_memory_tables`  | `read`      | List all memory tables accessible to this agent  |

Docs: https://km-alee.github.io/naqsha/tools/

## 9. Typed Event Bus

All 14 event types emitted by the Core Runtime:

| Event                   | Emitted when                                              |
|-------------------------|-----------------------------------------------------------|
| `RunStarted`            | Agent loop begins                                         |
| `AgentActivated`        | A specific agent starts its loop (multi-agent)            |
| `StreamChunkReceived`   | A chunk of the final answer is ready                      |
| `ToolInvoked`           | A tool call is dispatched                                 |
| `ToolCompleted`         | A tool call returns successfully                          |
| `ToolErrored`           | A tool call fails or is denied by policy                  |
| `SpanOpened`            | A new trace span is opened                                |
| `SpanClosed`            | A trace span closes with metrics                          |
| `BudgetProgress`        | Budget counters update (after each step/tool)             |
| `CircuitBreakerTripped` | Circuit Breaker threshold reached                         |
| `RunCompleted`          | Agent loop finishes with an answer                        |
| `RunFailed`             | Agent loop exits with a failure                           |
| `PatchMerged`           | A Reflection Patch was auto-merged                        |
| `PatchRolledBack`       | A previously merged patch was rolled back                 |

Subscribing:

```python
from naqsha import RuntimeEventBus
from naqsha.core.events import BudgetProgress, SpanClosed

bus = RuntimeEventBus()

@bus.subscribe
def on_budget(event: BudgetProgress):
    print(f"steps used: {event.steps_used}/{event.steps_limit}")

@bus.subscribe
def on_span_close(event: SpanClosed):
    print(f"span {event.span_id} closed: {event.token_count} tokens")
```

Docs: https://km-alee.github.io/naqsha/concepts/#typed-event-bus

## 10. Hierarchical QAOA Trace

**Q**uery → **A**ction → **O**bservation → **A**nswer. The canonical record of a run.

Every trace event (schema_version 2):

```json
{
  "schema_version": 2,
  "kind": "query | action | observation | answer | failure",
  "trace_id": "abc123",
  "span_id": "span_orch_001",
  "parent_span_id": null,
  "agent_id": "orch",
  "run_id": "abc123",
  "tool_name": "clock",
  "call_id": "c1",
  "payload": "2026-05-03T17:00:00+00:00",
  "ts": 1746291600.0
}
```

- `trace_id` = same as `run_id` for root spans
- V1 traces (missing `span_id`) are auto-upgraded on load
- `TraceStore` is append-only; no rewrite, no truncation

### Programmatic trace access

```python
from naqsha.tracing.store import JsonlTraceStore

store = JsonlTraceStore(".naqsha/traces")
trace = store.load_latest()
# trace.events → list of TraceEvent
# trace.trace_id, trace.query, trace.answer
```

### Replay against recorded observations

```python
from naqsha import build_trace_replay_runtime, load_run_profile
from naqsha.tracing.store import JsonlTraceStore

store = JsonlTraceStore(".naqsha/traces")
trace = store.load_latest()
runtime = build_trace_replay_runtime(trace, load_run_profile("local-fake"))
result = runtime.run(trace.query)
assert result.answer == trace.answer   # deterministic; no network calls
```

Docs: https://km-alee.github.io/naqsha/reference/tracing/

## 11. Dynamic Memory Engine

SQLite-backed (WAL mode) memory for Team Workspaces.

```python
from naqsha.memory import DynamicMemoryEngine

engine = DynamicMemoryEngine(".naqsha/memory.db")
shared  = engine.get_shared_scope()              # shared_* tables
private = engine.get_private_scope("researcher") # private_researcher_* tables
```

### DDL safelist

| Permitted                        | Forbidden (raises ForbiddenDDLError) |
|----------------------------------|--------------------------------------|
| `CREATE TABLE`                   | `DROP TABLE`                         |
| `CREATE INDEX`                   | `DROP INDEX`                         |
| `ALTER TABLE ADD COLUMN`         | `ALTER TABLE DROP COLUMN`            |
|                                  | `DELETE`, `TRUNCATE` as DDL          |

Regular DML (INSERT, SELECT, UPDATE, DELETE) is always permitted through MemoryScope directly.

### Shared memory

```python
# Create schema (DDL safelist enforced)
shared.execute("""
    CREATE TABLE IF NOT EXISTS shared_notes (
        id INTEGER PRIMARY KEY,
        agent_id TEXT NOT NULL,
        content TEXT NOT NULL,
        created_ts INTEGER NOT NULL DEFAULT (strftime('%s','now'))
    )
""")

# Write
shared.execute(
    "INSERT INTO shared_notes (agent_id, content) VALUES (?, ?)",
    ("researcher", "Finding: X implies Y"),
)

# Query
rows = shared.query("SELECT agent_id, content FROM shared_notes ORDER BY created_ts DESC")
```

### Token-budgeted retrieval

```python
from naqsha.memory import MemoryRetriever

retriever = MemoryRetriever(scope=shared, token_budget=512, table="shared_notes",
                            content_column="content")
results = retriever.retrieve("what implies Y")
# Returns UNTRUSTED EVIDENCE wrapped string; ranking = keyword_hits*1_000_000 + created_ts
```

### Team memory (multi-agent)

```python
from naqsha.memory.sharing import open_team_memory_engine, TeamMemoryConfig
from pathlib import Path

config = TeamMemoryConfig(db_path=Path(".naqsha/memory.db"))
engine = open_team_memory_engine(config)
# Wired automatically by build_team_orchestrator_runtime from [memory] in naqsha.toml
```

Docs: https://km-alee.github.io/naqsha/memory/

## 12. Multi-agent Team Workspaces

### naqsha.toml (complete example)

```toml
[workspace]
name         = "research-team"
orchestrator = "orch"
auto_approve = false

[memory]
db_path = ".naqsha/memory.db"

[reflection]
enabled          = true
auto_merge       = false   # ALWAYS false by default; opt-in only
reliability_gate = true
gate_paths       = ["tests/"]

[agents.orch]
role          = "orchestrator"
model_adapter = "openai_compat"
tools         = ["clock", "list_memory_tables"]
max_steps     = 30
max_tokens    = 8192

[agents.orch.openai_compat]
model       = "gpt-4o"
api_base    = "https://api.openai.com/v1"
api_key_env = "OPENAI_API_KEY"   # env var name — NEVER the key value

[agents.researcher]
role          = "worker"
model_adapter = "openai_compat"
tools         = ["clock", "read_file", "list_files", "list_memory_tables", "memory_schema"]
max_retries   = 3
max_steps     = 15
max_tokens    = 4096

[agents.researcher.openai_compat]
model       = "gpt-4o-mini"
api_base    = "https://api.openai.com/v1"
api_key_env = "OPENAI_API_KEY"
```

### Python API

```python
from pathlib import Path
from naqsha.orchestration.team_runtime import build_team_orchestrator_runtime
from naqsha.orchestration.topology import parse_team_topology_file
from naqsha import RuntimeEventBus

bus  = RuntimeEventBus()
topo = parse_team_topology_file(Path("naqsha.toml"))
rt   = build_team_orchestrator_runtime(topo, workspace_path=Path("."), event_bus=bus)
result = rt.run("Research and summarise topic X")
print(result.answer)
```

### Tool-Based Delegation — how it works

1. `build_team_orchestrator_runtime` auto-generates `delegate_to_<worker>` tools for each worker.
2. The orchestrator calls `delegate_to_researcher(task="...")`.
3. A new `CoreRuntime` is spawned with the worker's profile and a child `SpanContext`
   (same `trace_id`; new `span_id`; `parent_span_id` = orchestrator span).
4. The worker runs fully isolated — the orchestrator's `AgentContext` is NEVER passed.
5. The worker's answer is returned as a `ToolObservation` to the orchestrator.
6. If the worker's Circuit Breaker trips, the observation carries
   `metadata["kind"] = "TaskFailedError"` and `metadata["circuit_breaker"] = True`.

### Hierarchical trace fields

| Field           | Orchestrator     | Worker              |
|-----------------|------------------|---------------------|
| `trace_id`      | shared `run_id`  | shared `run_id`     |
| `agent_id`      | `"orch"`         | `"researcher"`      |
| `span_id`       | `"span_orch_..."` | `"span_researcher_..."` |
| `parent_span_id`| `null`           | `"span_orch_..."`   |

## 13. Model Adapters (NAP V2)

The Core Runtime only speaks **NAP V2**. Provider-specific formats live exclusively in adapter
files (`models/openai_compat.py`, `models/anthropic.py`, `models/gemini.py`, `models/ollama.py`).
There must be no `if provider == "openai"` checks outside these files.

### Supported adapters

| Adapter key      | Provider                                  | Notes                                  |
|------------------|-------------------------------------------|----------------------------------------|
| `fake`           | Built-in scripted model client            | No API keys; use for tests and CI      |
| `openai_compat`  | OpenAI, Azure, Together, Groq, …          | Any /chat/completions endpoint         |
| `anthropic`      | Anthropic Claude                          | ANTHROPIC_API_KEY                      |
| `gemini`         | Google Gemini                             | GOOGLE_API_KEY                         |
| `ollama`         | Local Ollama                              | base_url override for custom installs  |

### NAP V2 action structure

```python
from naqsha.models.nap import NAPAction, NAPAnswer, SpanContext

action = NAPAction(
    kind="action",
    calls=[{"id": "c1", "name": "clock", "arguments": {}}],
    span_context=span_ctx,  # optional; attaches active span
)
answer = NAPAnswer(kind="answer", text="done")
```

### Adding a new model adapter

1. Create `src/naqsha/models/<provider>.py`.
2. Implement `ModelClient` protocol: `next_message(messages, tools, *, span_context=None)`.
3. Register the adapter key in `models/factory.py` (`model_client_from_profile`).
4. Add corresponding toml sub-table support in `profiles.py`.
5. Write a test using the fake transcript replay fixture in `tests/models/`.

Docs: https://km-alee.github.io/naqsha/reference/models/

## 14. Reflection Loop and Rollback Manager

### Generate a patch

```bash
naqsha reflect --profile workbench <run_id>
# Creates an isolated workspace under .naqsha/reflection-workspaces/reflection-patch-<stamp>/
```

### Auto-merge (opt-in)

```toml
[reflection]
enabled          = true
auto_merge       = true          # opt-in; false everywhere by default
reliability_gate = true          # mandatory; auto_merge without this is silently ignored
gate_paths       = ["tests/smoke/"]
```

### How auto-merge + rollback works

1. Reliability Gate: runs `pytest` over `gate_paths` in a subprocess.
2. If pytest fails → patch is discarded; nothing is written.
3. If pytest passes → AutomatedRollbackManager:
   a. Snapshots `naqsha.toml` and workspace files to `.naqsha/backups/<stamp>/`.
   b. Applies the merge.
   c. Sets `boot_status = pending`.
4. On next `naqsha run` → boot probe verifies workspace health.
5. If boot probe fails → workspace is restored from snapshot; `PatchRolledBack` is emitted.
6. Manager prunes to 5 most recent backups.

### Python API

```python
from naqsha.reflection.loop import SimpleReflectionLoop

loop = SimpleReflectionLoop(project_root=Path("."))
patch = loop.run(run_id="<run_id>", profile=profile)
print(patch.ready_for_human_review)   # True if patch generated and gate passed
print(patch.auto_merged)              # True only if auto_merge=true AND gate passed
```

Docs: https://km-alee.github.io/naqsha/reflection/
Reference: https://km-alee.github.io/naqsha/reference/reflection/

## 18. Testing and CI patterns

### Run the test suite

```bash
uv run --extra dev pytest          # all tests; fake models; no API keys
uv run --extra dev pytest -x       # stop on first failure
uv run --extra dev pytest -k test_name   # filter by name
```

### Writing tests — key patterns

**Always use fake model clients:**

```python
from naqsha.models.trace_replay import TraceReplayModelClient
# or
# profiles: model_adapter = "fake" in naqsha.toml
```

**Tool tests:**

```python
from naqsha.tools import ToolExecutor, ToolRegistry, agent, AgentContext

registry = ToolRegistry()

@agent.tool(risk_tier="read", description="Test tool.")
def my_tool(value: str, ctx: AgentContext) -> str:
    return f"got: {value}"

registry.register(my_tool)
executor = ToolExecutor(registry)
ctx = AgentContext(agent_id="test", run_id="r1")
obs = executor.execute("my_tool", {"value": "hello"}, ctx)
assert obs.payload == "got: hello"
assert not obs.metadata.get("tool_error")
```

**Memory tests:**

```python
from naqsha.memory import DynamicMemoryEngine
import tempfile, pathlib

with tempfile.TemporaryDirectory() as tmp:
    engine = DynamicMemoryEngine(pathlib.Path(tmp) / "mem.db")
    shared = engine.get_shared_scope()
    shared.execute("CREATE TABLE IF NOT EXISTS shared_t (id INTEGER PRIMARY KEY, v TEXT)")
    shared.execute("INSERT INTO shared_t (v) VALUES (?)", ("hello",))
    rows = shared.query("SELECT v FROM shared_t")
    assert rows[0][0] == "hello"
```

**Event bus tests:**

```python
from naqsha import RuntimeEventBus
from naqsha.core.events import RunCompleted

received = []
bus = RuntimeEventBus()

@bus.subscribe
def capture(event: RunCompleted):
    received.append(event)

# wire bus into runtime and run...
assert len(received) == 1
```

### CI (`.github/workflows/ci.yml`)

- Runs on Python 3.11 and 3.12
- Steps: `uv sync --extra dev` → `ruff check .` → `pytest` → `mkdocs build --strict`
- No API keys; all tests use fake model clients
- Zero ruff errors required; zero pytest failures required

## 19. Extension points and first tasks

Good scoped tasks for adding to the codebase:

### Add a new @agent.tool to tools/starter.py

1. Open `src/naqsha/tools/starter.py`.
2. Add a function decorated with `@agent.tool(risk_tier=..., description=...)`.
3. Supported return types: `str`, `dict`, Pydantic BaseModel, or any type (converted to string).
4. Write a test in `tests/tools/`.
5. Add the tool name to relevant profiles / naqsha.toml examples.

### Add a new Model Adapter

1. Create `src/naqsha/models/<provider>.py`.
2. Implement `ModelClient`: `next_message(messages, tools, *, span_context=None) -> NAPAnswer`.
3. Register key in `models/factory.py`.
4. Add TOML config sub-table parsing in `profiles.py`.
5. Write a test in `tests/models/`.

### Add a new TUI panel

1. Create `src/naqsha/tui/panels/<panel>.py` with a Textual widget class.
2. Add colocated CSS in the widget's `DEFAULT_CSS` or a `.tcss` file.
3. Subscribe to the relevant event type(s) from the Typed Event Bus.
4. Register the panel in `src/naqsha/tui/app.py`.
5. Must meet the Workbench visual standard (tokyo-night theme, proper borders, readable markup).

### Expand the OWASP red-team corpus

1. Add scenarios to `tests/redteam/` following existing patterns.
2. Focus on: multi-agent delegation injection, Observation Sanitizer bypass attempts,
   memory namespace escape, DDL safelist circumvention.

### Improve Memory Schema Tool DDL error messages

1. Open `src/naqsha/memory/ddl.py`.
2. Improve `ForbiddenDDLError` messages to explain what is allowed and why the statement failed.
3. The Memory Schema Tool (`tools/starter.py`) calls this; better messages help the model adapt.
